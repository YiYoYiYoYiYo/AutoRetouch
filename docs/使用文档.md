# AI Beautify 使用文档

全自动 AI 修图与调色工作流——批量上传照片，AI 自动分析并生成专业修图建议，一键批量出片。

## 快速开始

### 1. 安装依赖

```bash
cd "AI Beautify"
pip install -r requirements.txt
# 或
pip install gradio Pillow numpy opencv-python requests colour-science
```

可选依赖（RAW 文件支持）：
```bash
pip install rawpy pillow-heif
```

### 2. 启动应用

```bash
python app.py
```

浏览器打开 `http://localhost:7860`

### 3. 使用流程

1. **上传照片** — 支持批量上传 JPEG/PNG/RAW
2. **填写场景描述** — 如"苏州园林旅行照，阴天"
3. **选择 VLM 后端** — 留空自动降级（推荐）
4. **点击"分析"** — AI 给出修图建议
5. **确认后点击"处理并导出"** — 批量出片

## VLM 后端说明

| 后端 | 模型 | 速度 | 质量 | 要求 |
|------|------|------|------|------|
| `glm` | GLM-4.1V-Thinking-Flash | ~11秒 | 最高 | 网络 + API Key |
| `agnes` | agnes-2.0-flash | ~5秒 | 高 | 网络 + API Key |
| `ollama` | qwen2.5vl:3b | ~3分钟 | 基础 | 本地 Ollama |

**自动降级顺序：** glm → agnes → ollama

**隐私开关：** 敏感照片可手动选择 `ollama` 后端，数据不出本机。

## 命令行使用

```python
from pipeline import BatchPipeline, load_images
from pathlib import Path

# 加载图片
images = load_images(["photo1.jpg", "photo2.jpg", "photo3.jpg"])

# 创建管线
pipe = BatchPipeline()

# 批量分析
suggestions = pipe.analyze_batch(images, context="苏州园林旅行照")

# 批量处理
batch = pipe.process_batch(images, suggestions)

# 批量导出
paths = pipe.export_batch(batch, Path("output"), fmt="jpeg")
```

## 参数规范

VLM 输出的修图参数遵循以下规范：

| 参数 | 范围 | 说明 |
|------|------|------|
| `exposure_ev` | -3.0 ~ +3.0 | 曝光值（EV） |
| `white_balance_k` | 2000 ~ 10000 | 色温（K） |
| `contrast` | -100 ~ +100 | 对比度 |
| `highlights` | -100 ~ +100 | 高光 |
| `shadows` | -100 ~ +100 | 阴影 |
| `saturation` | -100 ~ +100 | 饱和度 |

## 配置修改

编辑 `config.py` 修改：

- **API Key** — GLM 和 agnes 的密钥
- **默认后端** — `cfg.default_backend`
- **输出格式** — `cfg.processing.output_format`（jpeg/heic）
- **JPEG 质量** — `cfg.processing.jpeg_quality`
- **并发数** — `cfg.max_concurrent`

## 常见问题

**Q: 没有网络能用吗？**
A: 可以，选择 `ollama` 后端，需要先安装 Ollama 并拉取模型：`ollama pull qwen2.5vl:3b`

**Q: RAW 文件支持吗？**
A: 支持，需要安装 `rawpy`：`pip install rawpy`。RAW 文件会先转为 JPEG 再处理。

**Q: 处理速度太慢？**
A: 本地模型（ollama）处理一张约 3 分钟，云端约 11 秒。批量处理时使用云端可显著提速。

**Q: 如何切换输出格式？**
A: 界面上选择 jpeg 或 heic，或修改 `config.py` 中的 `cfg.processing.output_format`。
